📱 StayBusy iOS Design System

Complete UI kit with Dark & Warm themes, 40+ icons, and full component library

🎯 Overview

The StayBusy iOS Design System provides a comprehensive, accessible component library designed for trip planning and time-block management. It features two distinct visual themes optimized for different contexts and user preferences.

Dark Theme

Optimized for low-light environments. Features a warm coral-red accent (#E53935) with high contrast ratios and reduced eye strain.

View Dark Theme

Warm Theme

Friendly and approachable with vibrant warm tones (oranges, yellows, corals). Perfect for daytime use and social contexts.

View Warm Theme

Typography

All typography uses SF Rounded (Apple's rounded system font) with Dynamic Type support for accessibility.

  • Display (32px, Heavy): Large headings, countdown timers
  • Title Large (22px, Heavy): Screen titles, major section headers
  • Title (18px, Bold): Block titles, card headings
  • Body (16px, Medium): Main content, descriptions
  • Caption (12px, Medium, Monospaced): Time ranges, metadata, timestamps

Spacing (4pt scale)

xs: 4px | s: 8px | m: 12px | l: 16px xl: 24px | xxl: 32px

Shape

  • small (8px): Chips, small elements, input hints
  • medium (14px): Cards, buttons, modal corners
  • large (22px): Sheets, expanded surfaces, major containers

Interaction

  • Minimum tap target: 44×44pt (accessibility standard)
  • Pressed state opacity: 0.97 (subtle visual feedback)
  • Past block opacity: 0.4 (clear visual dimming)
  • Stroke width (current): 2pt (active block border)
  • Stroke width (open): 1.5pt dashed (open slot indicator)

🧩 Component Library

Ready-to-use components optimized for trip planning and time management.

Block Card

The core unit of the timeline. Displays activity blocks with category color stripe, title, time range, and location.

  • Default: Standard block with category stripe
  • Current: Accent border + glow, elevated surface
  • Past: Dimmed (40% opacity), no interaction
  • Open Slot: Dashed border, no fill, "OPEN — duration" label

Now/Next Bar

Pinned header showing current activity with countdown timer. Always displays a block—never empty.

  • Large countdown display (monospaced digits prevent jitter)
  • Category color stripe for visual context
  • Location and time range below title
  • Three states: current active, between blocks, day done

Buttons

  • Primary: Accent background, full width or auto
  • Secondary: Surface + border, for alternative actions
  • Ghost: Transparent, accent text, subtle actions
  • All buttons: 44pt minimum tap target, press feedback (0.97 opacity)

Category Chips

Toggle-able filters for category selection. Show color dot + label.

  • Unselected: border only, monochrome
  • Selected: accent background, white text/icon
  • Used in editor and filter UI

Input Fields

Text and time inputs with consistent styling and focus states.

  • Surface background with border
  • Focus state: accent border + subtle glow
  • Monospaced font for time fields
  • Placeholder text in tertiary color

♿ Accessibility

  • Contrast: All text on background passes 4.5:1 WCAG AA (minimum for small text)
  • Touch targets: Minimum 44×44pt for all interactive elements
  • Color is never the only signal: Current block has border treatment + color; categories show both color + icon
  • Dynamic Type: All text scales with system text size (SF Rounded Dynamic Type scales)
  • Monospaced digits: Timers and timestamps use monospaced numbers so they don't jitter as seconds update
  • Semantic HTML: Proper heading hierarchy, button roles, ARIA labels where needed

🛠️ Implementation Guide

CSS Variables (Theme-agnostic)

All components use CSS custom properties. Switch themes by changing the body class.

/* Dark theme (default) */ body.theme-dark { --bg: #0D0D0F; --surface: #17171A; --accent: #E53935; } /* Warm theme */ body.theme-warm { --bg: #FEF9F5; --surface: #FFFFFF; --accent: #FF6B35; }

Color Usage Pattern

Always reference theme colors through CSS variables, never hardcoded hex values:

/* ✓ Good */ background: var(--surface); color: var(--text-primary); /* ✗ Avoid */ background: #17171A; color: #F5F5F7;

Typography Pattern

Use semantic font sizes, never arbitrary pixel values:

font-family: var(--font-family-rounded); font-size: var(--font-size-title); font-weight: 700;

Spacing Pattern

Always use the 4pt spacing scale:

padding: var(--spacing-l) var(--spacing-xl); gap: var(--spacing-m); margin-bottom: var(--spacing-s);

Interactive States

All tappable elements need visual feedback:

button { transition: opacity 0.2s ease; } button:active { opacity: var(--opacity-pressed); /* 0.97 */ }

🚀 Quick Start

  1. Link tokens.css in your HTML <head>
  2. Set theme class on <body>: class="theme-dark" or class="theme-warm"
  3. Use CSS variables throughout your styles: background: var(--surface);
  4. Reference component showcase files for structure and patterns
  5. Icons are ready-to-use SVG with color inheritance from CSS variables
